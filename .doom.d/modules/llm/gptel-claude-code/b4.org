* Test 10: System prompt via --append-system-prompt
This tests that the =GPTEL_SYSTEM= property is correctly passed to Claude Code
as =--append-system-prompt= and affects the model's behavior.

:PROPERTIES:
:GPTEL_SYSTEM: You are a pirate captain named Blackbeard. You MUST speak entirely in pirate dialect. Every response must include "Arrr!" at least once. Never break character.
:END:

** Test 10b: Restrictive system prompt
:PROPERTIES:
:GPTEL_SYSTEM: You are a calculator. You can ONLY respond with numbers. No words, no explanations, just the numeric answer. If the input is not a math question, respond with "ERROR".
:END:

@user: What is 15 * 7?

** Test 10c: Empty system prompt
:PROPERTIES:
:GPTEL_SYSTEM:
:END:

@user: Say "hello" and nothing else.

** Verification
- [ ] Test 10a: Response is in pirate dialect, includes "Arrr!"
- [ ] Test 10b: Response is just "105" (or the number, no words)
- [ ] Test 10c: Behaves normally with empty/no system prompt
- [ ] =*Messages*= shows =--append-system-prompt= with the custom prompt text for 10a and 10b
- [ ] =*Messages*= does NOT show =--append-system-prompt= for 10c (empty prompt skipped)
- [ ] System prompt does not appear in the buffer output (only affects behavior)

* Test 11: Concurrent queries
This tests that two simultaneous Claude Code processes can run without
interfering with each other. Each gets its own process, session-id, and
MCP routing.

** Query A
Send this FIRST, then IMMEDIATELY switch to Query B heading and send there
before Query A finishes.

Tip: Use =M-x gptel-send= here, then quickly =C-c C-n= (or navigate) to
Query B heading and =M-x gptel-send= there.

@user: Count slowly from 1 to 20. Put each number on its own line. Take your time.

** Query B
Send this while Query A is still running.

@user: List the first 10 elements of the periodic table, one per line, with their atomic numbers.

- [ ] Both responses completed without corruption (no interleaved text)
- [ ] Query A shows numbers 1-20
- [ ] Query B shows elements H through Ne with atomic numbers

- [ ] =gptel--request-alist= is empty after both complete
